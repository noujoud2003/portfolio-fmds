Prédiction du Remaining Useful Life (RUL)
Batteries Lithium-ion 18650

Étude réalisée par Noujoud Ouni

Contexte du dataset

Dataset issu de l’Hawaii Natural Energy Institute :
• 14 cellules NMC-LCO 18650 (capacité nominale 2.8 Ah)
• Cyclées à 25°C, plus de 1000 cycles
• Charge CC-CV à C/2 – Décharge à 1.5C
• Objectif : prédire le RUL restant (en heures) à partir de 7 features extraites des cycles

Pourquoi le modèle LSTM est-il le meilleur ?

Justification principale :
Le LSTM obtient les meilleurs compromis sur les métriques clés (R², RMSE, MAE) tout en capturant très bien la dynamique temporelle non linéaire de dégradation en fin de vie, contrairement aux modèles arbres (XGBoost, RF) qui surestiment systématiquement le RUL restant quand il approche 0.
  • R² test : 0.9722 – 0.9753 (très élevé)
  • RMSE test : ~0.0138–0.0146 h (~50–52 secondes)
  • MAE test : ~0.010–0.011 h (~36–40 secondes)
  • Meilleure capacité à suivre la pente réelle sur les 3 batteries test
  • Early stopping efficace → évite sur-apprentissage visible après epoch 11–15

Architecture du modèle LSTM retenu

Modèle final retenu après plusieurs itérations (dropout réduit, patience augmentée, scaling cohérent).
  • Couche 1 : LSTM 64 unités, activation ReLU, input (10, 7)
  • Dropout : 0.2 (ou 0.1 dans les meilleures versions)
  • Couche Dense : 32 unités, ReLU
  • Dropout : 0.2
  • Sortie : Dense 1 (linéaire) → RUL restant en heures
  • Optimiseur : Adam
  • Loss : MSE
  • Métrique : MAE
  • Early Stopping : patience=10, restore_best_weights=True
  • Nombre d’époques max : 50
  • Batch size : 32

Limites & Perspectives

Limites observées :
  • Surestimation systématique du RUL en toute fin de vie (RUL < 0.05 h)
  • Validation loss parfois instable après le meilleur epoch
  • Seulement 3 batteries en test → généralisation à confirmer sur plus de cellules
  • Pas de prise en compte explicite de la température ou de la SOH réelle
Perspectives d’amélioration :
  • Ajouter Bidirectional LSTM ou Transformer pour mieux capturer les dépendances longues
  • Utiliser Huber loss (plus robuste aux outliers en fin de vie)
  • Intégrer features physiques (température, impedance, SOH estimé)
  • Essayer uncertainty quantification (Monte-Carlo Dropout, Bayesian NN)
  • Passer à un split leave-one-out pour une meilleure évaluation de généralisation
  • Déployer le modèle dans un dashboard interactif (Streamlit / Dash)
