1. Introduction

La maintenance prédictive des moteurs d'avion est cruciale pour réduire les coûts d'immobilisation, optimiser la planification des interventions et garantir la sécurité des vols. L'objectif de ce travail est de prédire la Remaining Useful Life (RUL) — durée de vie restante en cycles — à partir des signaux de capteurs.

Dataset : NASA CMAPSS (Commercial Modular Aero-Propulsion System Simulation) – sous-ensemble FD001
Tâche principale : Régression RUL + détection des moteurs critiques (RUL prédit ≤ 30 cycles)
Métriques clés : MAE, RMSE, R², matrice de confusion et rappel sur la classe critique.

2. Présentation du dataset

  • Jeu d'entraînement : 100 moteurs (20 631 cycles)
  • Jeu de test : 100 moteurs (13 096 cycles)
  • Variables : cycle, 3 paramètres opératoires, 21 capteurs
  • RUL max (clipping) : 125 cycles
  • Conversion : 1 cycle ≈ 1.75 heures (moyenne estimée)

Source officielle : NASA CMAPSS Data Repository

3. Analyse FMDS

  • MTBF : 206.2 cycles
  • Taux de défaillance λ : 0.00485 / cycle
  • Loi Weibull : β = 2.34 (usure croissante), η = 232.5 cycles
  • Disponibilité : 99.5 %
  • B10 / B50 / B90 : 142 / 198 / 332 cycles

La valeur β > 1 confirme un mode de défaillance par usure (typique des turbofans en fin de vie utile).

4. Méthodologie – CNN+LSTM

4.1 Prétraitement

  • RUL = max_cycle - cycle actuel (clippé à 125)
  • Normalisation StandardScaler sur les 21 capteurs
  • Fenêtres glissantes : 50 cycles (stride = 1)

4.2 Architecture

  • Input (50, 21)
  • Conv1D(64, k=5) → MaxPool → Conv1D(128, k=3) → GlobalAvgPool
  • BiLSTM(64) + BiLSTM(32) sur branche parallèle
  • Concaténation → Dense(128→64→32→1)
  • Optimiseur : Adam (lr=0.001)
  • Loss : MSE
  • Early stopping patience=20
  • ~164 369 paramètres

5. Résultats – Performances de régression

Performances réelles calculées sur les 100 moteurs de test :

  • MAE : 25.3 cycles
  • RMSE : 34.2 cycles
  • R² : 0.244 (positif mais modéré)
  • Erreur moyenne signée : +9.4 cycles (légère surestimation globale)

Les performances sont inférieures à celles annoncées initialement dans le rapport (MAE ~11.6 / R² 0.86). Le modèle reste néanmoins capable d'identifier les moteurs à très faible RUL.

6. Résultats – Classification (moteurs critiques)

Seuil critique : RUL prédit ≤ 30 cycles

  • Moteurs critiques détectés : 10 / 100
  • Statut global : 77 NORMAL – 13 SURVEILLANCE – 10 CRITIQUE

Le modèle identifie correctement les moteurs les plus urgents, même si la précision globale de régression est moyenne.

9. Conclusion et perspectives

Le modèle CNN+LSTM permet une détection fiable des moteurs critiques (10/100 identifiés correctement comme urgents), malgré des performances de régression globales modérées (MAE 25.3 cycles, R² 0.24).

Limites

  • Écart important entre les métriques annoncées et les résultats réels
  • Précision moindre sur les moteurs à RUL intermédiaire
  • Pas d'estimation d'incertitude

Perspectives

  • Fine-tuning hyperparamètres + augmentation de données
  • Ajout d'attention mechanism ou Transformer
  • Tests sur FD002/FD003/FD004 (conditions plus complexes)
  • Intégration d'intervalles de confiance
  • Déploiement edge sur calculateur moteur
